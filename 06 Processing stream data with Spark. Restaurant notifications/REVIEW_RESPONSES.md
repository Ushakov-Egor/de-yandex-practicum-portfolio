# Ответы на замечания ревьювера

---

### 1. Убрать автозапуск стрима при импорте файла

Весь исполняемый код (инициализация SparkSession, чтение из Kafka, запуск стрима и `awaitTermination()`) обёрнут в блок `if __name__ == "__main__":`. Теперь при импорте файла выполняются только определения функций и констант — никаких побочных эффектов.

---

### 2. Сделать нормальную устойчивость к рестартам через checkpoint

В `.writeStream` добавлен `.option("checkpointLocation", "/tmp/checkpoints/restaurant_subscribe")`. При рестарте job Spark восстанавливает офсеты из checkpoint и продолжает обработку с того места, где остановился — сообщения, пришедшие пока сервис лежал, не теряются.

> Текущий путь `/tmp` подходит для учебного окружения. В продакшне checkpoint следует хранить в постоянном хранилище (HDFS, S3), так как содержимое `/tmp` очищается при перезагрузке контейнера.

---

### 3. Обеспечить идемпотентность записи

Запись в PostgreSQL переведена с `mode("append")` на `INSERT ... ON CONFLICT DO NOTHING` по бизнес-ключу `(restaurant_id, adv_campaign_id, client_id)`. При повторной обработке батча после рестарта дубли в таблицу не попадут. Для работы механизма требуется уникальный индекс — SQL для его создания добавлен в README.md.

---

### 4. Починить вычисление trigger_datetime_created

Заменили `F.lit(int(datetime.now().timestamp()))` на `F.unix_timestamp()`. Старый вариант вычислялся на драйвере один раз при построении DataFrame и оставался константой на весь аптайм процесса. Теперь значение вычисляется Spark-ом на каждый батч в UTC, независимо от таймзоны Python-процесса. Импорт `datetime` удалён как ставший ненужным.

---

### 5. Добавить фильтрацию активных кампаний

В функцию `join()` добавлен `.filter()` по условию:

```
adv_campaign_datetime_start <= текущее время <= adv_campaign_datetime_end
```

Текущее время берётся через `F.unix_timestamp()` — тот же механизм, что и для `trigger_datetime_created`, всегда в UTC. В обработку проходят только активные на момент батча кампании.

---

### 6. Развязать конфиг от localhost и хардкоженных секретов

Все чувствительные параметры (хосты, порты, логины, пароли, топики Kafka) вынесены в переменные окружения через `os.environ.get()`. В репозиторий добавлены:
- `.env.example` — шаблон с незаполненными ключами для других разработчиков
- `.gitignore` — реальный `.env` с секретами исключён из коммитов

---

README.md полностью переписан под текущее состояние кода:
- актуальная схема архитектуры
- реальная структура репозитория (`src/scripts/` вместо устаревших `src/dags` и `src/sql`)
