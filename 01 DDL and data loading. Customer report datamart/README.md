# DDL и загрузка данных. Витрина клиентских отчётов.

## Описание задачи

Маркетплейс товаров ручной работы хранит данные в нескольких источниках с разными схемами. Нужно объединить данные из всех источников в единую звёздную схему DWH и построить аналитическую витрину с ежемесячными метриками по каждому заказчику: сумма трат, количество заказов, любимая категория товаров, медианное время выполнения заказа, топ-мастер.

## Что было сделано

- Разработан DDL таблицы `customer_report_datamart` с 20+ метриками на заказчика за отчётный период
- Написан скрипт миграции данных из 4 разнородных источников (`craft_market_wide`, `craft_market_masters_products`, `craft_market_orders_customers`, `craft_market_craftsmen`) в нормализованные таблицы DWH (`d_customer`, `d_craftsman`, `d_product`, `f_order`)
- Реализована инкрементальная загрузка через CTE: отслеживаются изменения по всем измерениям с момента последней загрузки
- Разделена логика вставки новых записей (NEW) и обновления существующих (UPDATE) через детектирование дельты

## Стек и инструменты

`SQL` `PostgreSQL` `DDL` `DML` `CTE` `Оконные функции` `Звёздная схема` `Инкрементальная загрузка`

## Структура папки

- `scripts/` — SQL-скрипты для DDL и загрузки данных
  - `DDL_customer_report_datamart.sql` — определение таблицы витрины
  - `DDL_load_dates_customer_report_datamart.sql` — таблица метаданных для отслеживания времени загрузки
  - `SQL_load_from_old_and_new_sources.sql` — миграция из нескольких источников в DWH
  - `SQL_incremental_load_customer_report_datamart.sql` — логика инкрементального обновления витрины
